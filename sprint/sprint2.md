# Sprint 2 명세

## 목표

계좌별 목표 포트폴리오를 설정하고, 현재 보유 상태와의 괴리를 시각화한다.

---

## 화면 목록

| 경로                          | 화면명          | 주요 기능                                  |
|-----------------------------|--------------|----------------------------------------|
| `/accounts/[id]`            | 계좌 상세 (수정)   | "목표 포트폴리오 보기 →" 링크 추가                  |
| `/accounts/[id]/portfolio`  | 목표 포트폴리오     | 포트폴리오 설정 여부에 따라 빈 상태 또는 괴리 분석 테이블 표시   |

편집 및 삭제는 포트폴리오 페이지 내 모달로 처리한다.

---

## API 명세

### 공통

- Base URL: `/api/v1`
- Content-Type: `application/json`
- 에러 응답: `{ "error_message": "메시지" }` + HTTP 상태코드

---

## 포트폴리오

### 포트폴리오 응답 객체 (`Portfolio`)

```json
{
  "account_id": 1,
  "description": "바이앤홀드 전략 — 글로벌 분산 장기투자",
  "items": [
    {
      "asset": {
        "ticker": "379810",
        "name": "KODEX 미국나스닥100",
        "type": { "code": "DOMESTIC_STOCK", "label": "국내주식" },
        "currency_exposure": true
      },
      "target_ratio": 30.0
    }
  ]
}
```

| 필드          | 타입              | 설명             |
|-------------|-----------------|----------------|
| account_id  | number          | 소속 계좌 ID       |
| description | string?         | 투자 철학 (선택)     |
| items       | PortfolioItem[] | 목표 종목 및 비율 목록  |

**PortfolioItem**

| 필드           | 타입     | 설명                     |
|--------------|--------|------------------------|
| asset        | Asset  | 자산 정보                  |
| target_ratio | number | 목표 비율 (%, 소수점 1자리)     |

---

### `GET /api/v1/accounts/{accountId}/portfolio` — 포트폴리오 조회

**Response 200**: `Portfolio` 객체
**Response 404**: 포트폴리오 미설정 시

---

### `POST /api/v1/accounts/{accountId}/portfolio` — 포트폴리오 생성

**Request Body**

```json
{
  "description": "바이앤홀드 전략 — 글로벌 분산 장기투자",
  "items": [
    { "asset_ticker": "379810", "target_ratio": 30.0 },
    { "asset_ticker": "276000", "target_ratio": 20.0 }
  ]
}
```

| 필드           | 필수 | 설명                         |
|--------------|----|----------------------------|
| description  | -  | 투자 철학 (선택)                 |
| items        | ✓  | 1개 이상, 비율 합계 100% 이어야 함    |
| asset_ticker | ✓  | 자산 코드                      |
| target_ratio | ✓  | 목표 비율 (양수)                 |

**Response 201**: 생성된 `Portfolio` 객체
**Response 400**: 비율 합계 != 100% 또는 유효하지 않은 자산 코드
**Response 404**: 계좌 없음
**Response 409**: 이미 포트폴리오가 존재함

---

### `PUT /api/v1/accounts/{accountId}/portfolio` — 포트폴리오 수정

items 전체 교체 방식 (PATCH 아님).

**Request Body**: `POST` 와 동일한 구조

**Response 200**: 수정된 `Portfolio` 객체
**Response 400**: 비율 합계 != 100%
**Response 404**: 계좌 또는 포트폴리오 없음

---

### `DELETE /api/v1/accounts/{accountId}/portfolio` — 포트폴리오 삭제

**Response 204**
**Response 404**: 계좌 또는 포트폴리오 없음

## 괴리 분석

### 괴리 분석 응답 객체 (`PortfolioGap`)

```json
{
  "total_current_value": 10000000,
  "needs_rebalancing": true,
  "items": [
    {
      "asset": {
        "ticker": "379810",
        "name": "KODEX 미국나스닥100",
        "type": { "code": "DOMESTIC_STOCK", "label": "국내주식" },
        "currency_exposure": true
      },
      "target_ratio": 30.0,
      "current_ratio": 22.5,
      "gap": -7.5,
      "target_amount": 3000000,
      "current_amount": 2250000
    }
  ]
}
```

| 필드                   | 타입               | 설명                                      |
|----------------------|------------------|-----------------------------------------|
| total_current_value  | number           | 계좌 전체 평가금액                              |
| needs_rebalancing    | boolean          | 어느 종목이든 \|gap\| > 5% 이면 true            |
| items                | GapItem[]        | 보유 종목 + 포트폴리오 종목의 합집합 (중복 제거)           |

**GapItem**

| 필드             | 타입      | 설명                                        |
|----------------|---------|-------------------------------------------|
| asset          | Asset   | 자산 정보                                     |
| target_ratio   | number? | 목표 비율. `null` = 포트폴리오에 없지만 보유 중인 종목      |
| current_ratio  | number? | 현재 비율. `null` = 포트폴리오에 있지만 미보유 종목        |
| gap            | number? | `target_ratio - current_ratio`. 둘 다 있을 때만 |
| target_amount  | number? | `total_current_value × target_ratio / 100` |
| current_amount | number? | 현재 평가금액                                   |

---

### `GET /api/v1/accounts/{accountId}/portfolio/gap` — 괴리 분석

**Response 200**: `PortfolioGap` 객체
**Response 404**: 계좌 또는 포트폴리오 없음

### 화면 2: 목표 포트폴리오 (`/accounts/[id]/portfolio`)

#### 2-A: 포트폴리오 미설정 상태

```
┌─────────────────────────────────────────────┐
│  목표 포트폴리오                              │
│                                             │
│  아직 목표 포트폴리오가 없습니다.               │
│  투자 전략을 설정하고 현재 상태와 비교해보세요.  │
│                                             │
│          [ 포트폴리오 설정하기 ]              │
└─────────────────────────────────────────────┘
```

#### 2-B: 포트폴리오 설정 후

```
┌─────────────────────────────────────────────────────────────┐
│ ← ISA 계좌                          [ 편집 ] [ 삭제 ]       │
│                                                             │
│ 투자 철학                                                    │
│ "바이앤홀드 전략 — 글로벌 분산 장기투자"                      │
│                                                             │
│ ✅ 포트폴리오가 목표에 근접합니다.                            │
├──────────────┬──────┬──────┬────────┬──────────┬──────────┤
│ 종목          │ 목표% │현재% │  괴리  │  목표금액  │  현재금액  │
├──────────────┼──────┼──────┼────────┼──────────┼──────────┤
│ KODEX 나스닥  │  30% │ 28%  │  -2%  │ 300만    │  280만   │
│ KODEX S&P500 │  20% │ 21%  │  +1%  │ 200만    │  210만   │
│ ACE KRX금현물│  10% │  -   │  -10% │ 100만    │    -     │ ← 미보유
│ TIGER CD금리 │   -  │ 15%  │    -  │    -     │  150만   │ ← 포폴 외 보유
└──────────────┴──────┴──────┴────────┴──────────┴──────────┘
```

괴리 표시 규칙:
- `|gap| ≤ 5%`: 기본색 (정상)
- `gap < -5%`: 빨간색 + "부족" 뱃지 (매수 필요)
- `gap > +5%`: 주황색 + "초과" 뱃지
- `current_ratio`가 null (미보유): 행 배경 강조
- `target_ratio`가 null (포트폴리오 외 보유): 이탤릭 처리

#### 2-C: 리밸런싱 필요 시 — 배너 추가 (2-B 위에 삽입)

```
┌─────────────────────────────────────────────────────────────┐
│ ⚠️  리밸런싱이 필요합니다                                     │
│ 일부 종목이 목표 비율과 5% 이상 차이납니다.                    │
│ • KODEX 미국나스닥100: 750,000원 추가 매수 필요               │
│ • TIGER CD금리: 목표 비율 미설정 (포트폴리오 외 보유)          │
└─────────────────────────────────────────────────────────────┘
```
